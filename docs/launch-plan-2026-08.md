# 8월 공개 준비 계획 (2026-08 Launch Plan)

> 작성: 2026-07-24 | 상태: 초안(승인 대기)
> 두 트랙을 다룬다: ① 공개 전 미비점 마감(launch readiness), ② BomLens 9차 파리티 흡수.
> 로드맵 단일 진실은 `docs/completeness-master-plan.md`이며, 이 문서는 8월 공개라는
> 마일스톤 관점에서 잔여 항목을 재배열한 실행 계획이다. 완료 시 상태 열을 갱신한다.

## 0. 현재 상태 진단 (2026-07-24)

- 코드: v0.14.0 태그 이후 v0.15~v0.17 항목이 전부 main에 무태그로 머지됨 (마지막 PR #510).
- 문서: 2026-07 문서 품질 캠페인(P1~P3) 종결. CRA 매핑 문서(X2)까지 반영.
- 저장소: 이미 public. 커뮤니티 파일(README, CONTRIBUTING, SECURITY, CODE_OF_CONDUCT, LICENSE, 이슈 템플릿) 완비.
- **최대 미비점은 운영 트랙 O 전체가 미착수라는 것.** 데모 SaaS는 미배포 상태이고,
  `deploy-hetzner.yml`은 v0.11.0~v0.13.0 모든 태그에서 12초 만에 실패해 왔다
  — 원인은 코드가 아니라 Hetzner 서버 미프로비저닝 + GitHub `demo` Environment 시크릿 미등록(운영자 1회 설정).
- BomLens: 파리티 기준점 `6b2ca03`(8차, 2026-07-18) 이후 64커밋·v1.8.3 릴리즈가 쌓임. 9차 리뷰 필요.

### R2. 공개 버전·태그 전략 (사용자 결정 필요)

- v0.15~v0.17이 무태그 마일스톤으로 main에 누적된 상태다. 공개 시점에 릴리즈 태그가 필요하다.
- 결정 사항: **8월 공개 태그를 v1.0.0으로 할지, v0.x를 유지할지.**
  - v1.0.0 안: "Black Duck/Snyk 수준" 포지셔닝과 공개 발표에 부합. 단 API 안정성 약속이 따라옴.
  - v0.18.0 안: 보수적. 공개 후 피드백 반영 뒤 v1.0.0.
- 태그 컷 시 릴리즈 노트에 v0.15~v0.17 무태그 분량을 소급 정리한다 (EN/KO).
- 릴리즈 게이트(draft→검증→publish)는 v0.14.0에서 재시도 없이 통과 실적 있음 — 그대로 사용.

### R3. 신규 사용자 관문 리허설 (launch rehearsal)

공개 직후 유입될 신규 사용자의 3개 진입로를 실제로 밟아 본다.

- R3-1: 클린 환경에서 `install.sh` quickstart 리허설 (docs-uat 단언과 실물 대조. R1-5와 묶어 진행)
- R3-2: README → 문서 사이트 → 데모 링크 정합 점검 (죽은 링크·구버전 문구·스크린샷 표류)
- R3-3: 데모 read-only 시나리오 점검 — 시드 데이터(`seed --demo-only`)가 보여 줄 만한 상태인지
  (프로젝트 구성, 취약점·라이선스 분포, Compare·그래프·검색이 시연 가능한 데이터인지)

### R4. 보안·품질 부채 중 공개 전 가치 항목 (§7에서 선별)

| # | 항목 | 근거 | 규모 |

### R5. 기능 백로그 중 공개 가치 항목

- **X1 취약점 SLA/aging 추적** (마스터플랜 §9 최우선 후보): `first_detected_at` 스키마 확장(expand)
  + aging 뷰 + severity별 SLA 기한·초과 알림. 중간 규모. **8월 공개 태그에 포함 목표.**
  CRA 문서(X2)가 "지체 없는 수정" 의무의 제품 근거로 이 항목을 한계로 명시하고 있어, 공개 전 마감의 서사 가치가 크다.
- X3 OSS 건강도 신호: 외부 피드 의존이 커 공개 전 범위에서 제외(백로그 유지).
