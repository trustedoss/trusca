# Design Concept: TrustedOSS Portal

**컨셉명**: Calm Compliance  
**최초 작성**: 2026-05-04

---

## 한 줄 정의

> **"소음 없이 위험을 보여주는 도구"**  
> 법무팀이 처음 열었을 때 훈련 없이 빨간 것이 문제라는 걸 바로 알 수 있어야 한다.

---

## 핵심 철학

**데이터가 주인공이다.**  
버튼, 그래디언트, 장식 요소는 눈에 띄지 않는다. 화면을 열었을 때 시선이 자연스럽게 데이터와 리스크 상태로 향해야 한다. 컴플라이언스 도구의 본질은 "이 패키지가 안전한가"라는 질문에 즉시 답하는 것이다.

---

## 색의 역할 — 리스크 언어

UI에서 색은 오직 하나의 목적만 갖는다. **색 = 정보**이지 분위기가 아니다.

| 색 | Hex | 의미 | 사용처 |
|----|-----|------|--------|
| **초록** | `#0ecb78` | 신뢰 · 허용 · 완료 | Primary 버튼, 허용 배지, 리스크 바, 완료 상태 |
| **노랑** | `#f59e0b` | 주의 · 조건부 검토 필요 | 조건부 배지, 분포 바 황색 구간 |
| **빨강** | `#ef4444` | 위험 · 금지 · 오류 | 금지 배지, 행 배경 틴트, 오류 상태 |
| **회색 (neutral)** | `#71717a` 계열 | 나머지 모든 것 | 텍스트, 테두리, 배경, 아이콘 |

**규칙**: 초록·노랑·빨강은 리스크/상태 표현 이외의 용도로 사용하지 않는다. Primary 버튼의 초록은 "이 액션은 안전하다"는 의미를 겸한다.

---

## 밀도 — Compact Enterprise

Linear, Jira, GitHub와 동일한 정보 밀도를 유지한다.  
패딩은 충분하되, 스크롤 없이 한 화면에서 많은 패키지를 볼 수 있어야 한다.  
법무/IPR 담당자가 수십 개 패키지를 검토할 때 스크롤을 최소화하는 것이 목표다.

- 행 높이: `py-3` (48px 내외)
- 카드 패딩: `p-4`
- 사이드바 너비: `224px` (14rem)
- 컨텐츠 좌우 패딩: `px-6`

---

## 타이포그래피 — 역할 분리

| 폰트 | 용도 |
|------|------|
| **Inter** | 모든 UI 텍스트 — 레이블, 버튼, 설명, 제목 |
| **JetBrains Mono** | 패키지명, 버전, ID, 해시처럼 "값"인 것들 |

기술적 데이터(패키지명, 버전)를 일반 텍스트와 폰트로 구분함으로써 별도 색상 없이도 시각적 계층이 생긴다.

---

## 컴포넌트 원칙

### 배지 (Badge / Chip)
- 형태: `rounded-full` pill 형태
- 리스크 4단계: 허용(초록) · 조건부(노랑) · 금지(빨강) · 미식별(회색)
- 텍스트 앞에 기호 접두사 사용: `✓ 허용` · `⚠ 조건부` · `⊘ 금지` · `? 미식별`

### 테이블 행의 리스크 인디케이터
- 각 행 좌측에 `w-1` 세로 컬러 바로 리스크를 즉시 스캔 가능하게 표시
- 금지 항목 행은 배경에 `#fff8f8` 틴트 적용 — 배지 없이도 위험 행임을 인지

### 모달 / 드로어
- 모달: 삭제 확인 등 단순 yes/no → `AlertDialog`
- 드로어: 취약점 상세처럼 내용이 많고 컨텍스트를 유지해야 할 때 → `Sheet` (우측 슬라이드)

### 빈 상태 (Empty State)
- 스캔 전, 결과 없음 등 빈 상태는 텍스트만으로 처리. 일러스트레이션 사용 금지.
- 예: "스캔이 완료되면 BOM이 여기에 표시됩니다."

### 로딩 상태
- 테이블: 스켈레톤 행 (shimmer 애니메이션)
- 버튼 액션: 버튼 내 스피너 + 비활성화 (별도 오버레이 금지)

---

## 다크모드

다크모드를 지원하되, **기본값은 라이트 모드**다.  
법무/IPR팀의 업무 환경(사무실, 외부 미팅)을 고려하면 라이트 모드가 가독성이 높다.  
CSS 변수 토큰(`--background`, `--foreground`, `--primary` 등)으로 테마를 전환한다.

---

## 하지 않는 것

- 그래디언트 배경, 글로우 효과, 블러 오버레이
- 리스크 이외 목적의 색상 사용 (ex. 섹션 구분에 색 사용)
- 일러스트레이션, 애니메이션 마스코트
- 3단 이상의 계층적 카드 중첩
- 모바일 전용 레이아웃 (반응형은 지원하나, 데스크톱 우선)
