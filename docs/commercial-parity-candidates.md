# 상용 제품 격차 후보 (Commercial Parity Candidates)

상용 오픈소스 관리 제품(Black Duck, 엘에스웨어 포세라 위드 블랙덕)의 공개 기능을 TRUSCA 현재 구현과 대조해, 도입 가치가 있는 격차를 후보로 추적하는 문서다. 자매 오픈소스 BomLens 기준 격차는 [`bomlens-parity-review.md`](./bomlens-parity-review.md)가 따로 다룬다 — 이 문서는 상용 기준이다.

출처: 보안뉴스 ISEC 2026 미리보기(2026-07-08, `idx=144551`), 엘에스웨어 포세라 위드 블랙덕 기능 고도화 보도(2026-07), Black Duck SCA 2026.1 제품 자료.

## 판정 원칙

1. TRUSCA 역할(영속 거버넌스 포털)에 맞는 격차만 후보로 채택한다. 아티팩트 저장소·호스트 런타임 에이전트로의 확장은 역할 경계로 제외한다(포세라의 "오픈소스 저장소 운영", "서버 운영단계 상시 점검"이 여기 해당).
2. 이미 구현된 기능은 후보가 아니다. 대조 결과 포세라/Black Duck 헤드라인 기능 다수(CVSS/EPSS/KEV 우선순위, ML-BOM 수용, 다포맷 SBOM 수출, CI/CD 자동점검·빌드 차단)는 TRUSCA가 이미 보유한다.

## C1 — 라이선스·취약점 콘텐츠 한글화

핵심: 지금은 화면 문자열만 EN/KO(react-i18next)이고, 정작 라이선스 전문(`apps/backend/services/license_texts/*.txt`)은 영어 SPDX 원문, CVE 상세 설명은 Trivy/NVD 영어 원문이다. UI i18n과 콘텐츠 번역은 별개 문제다.

상용 근거: 포세라 기능 고도화 보도가 "라이선스 전문, 취약점 상세 정보 등을 한글로 제공"을 신기능으로 명시했다. 주 고객이 국내 금융권(신한금융그룹·현대캐피탈·우리은행)이며, 콘텐츠 한글화가 이들의 실제 요구다. TRUSCA는 한국어 GA 동시 출시를 표방하므로 시장 차별화 지점이다.

범위 제안(단계적):
- (a) 라이선스 요약·의무사항 KO 번역 — 카탈로그 52종은 유한집합이라 사람 번역이 현실적이다. 의무사항 카탈로그(`obligation_catalog.py`)와 분류 판정 문구부터 시작한다.
- (b) CVE 상세 설명 KO — 대량·동적이라 기계번역 또는 지연 번역이 필요하다. 우선순위 높은 CVE(KEV·Critical)부터 선별 적용을 검토한다.

주의: 라이선스 전문 자체는 법적 정본이 영어다. 번역본은 참고용임을 명시하고 원문을 병기한다(정본=영어 원칙 유지). NOTICE 번들에 들어가는 전문은 영어 유지.

## C3 — 정책 반영 맞춤 SBOM 프로파일

핵심: SBOM 수출 4포맷(`sbom_export.py`), 라이선스 정책 엔진(`license_policy_service.py`), VEX 임베드가 이미 있다. 빠진 것은 "조직 정책을 반영·주석한 맞춤 SBOM 프로파일" 출력이다.

상용 근거: 포세라가 "블랙덕 엔진에서 선택한 정책에 따라 조직의 보안 정책과 오픈소스 사용 기준을 반영한 맞춤형 SBOM 출력" 기능을 추가했다. Black Duck 2026.1도 SBOM visibility·정책 연동을 강조한다.

범위 제안: 기존 수출·정책 자산을 재사용하는 작은 확장이다. 수출 시 정책 프로파일을 선택하면 위반 컴포넌트를 CycloneDX `properties`/SPDX annotation으로 주석하거나, 프로파일 규칙에 따라 필터한다(예: 금지 라이선스 강조, 정책 위반 표식, 조직 프로파일별 출력). 저비용·차별화 항목이라 C1 다음의 실용 후보다.

## C4 — 다언어 도달성 분석 확대 (BomLens 협업)

핵심: 현재 도달성 분석은 Go(`integrations/govulncheck.py`, `tasks/scan_reachability.py`)만 지원한다. Black Duck의 reachability는 다언어(Java·JavaScript 등)로 실제 호출되는 코드에 한정해 findings를 70~95% 줄인다 — 이것이 상용 대비 TRUSCA의 최대 격차다.

협업 근거: 도달성 산출(언어별 call-graph 구성)은 단발 스캔 도구의 강점 영역이라 BomLens(`sktelecom/sbom-tools`) 역할에 가깝고, TRUSCA는 그 결과를 ingest해 findings 필터·우선순위로 표면화하는 소비자에 가깝다. 따라서 이 후보는 TRUSCA 단독이 아니라 BomLens와 공동 설계로 진행한다.

역할 분담(초안):
- BomLens: 언어별 도달성/call-graph 산출을 스캔 산출물에 포함(단발 스캔 강점 확장).
- TRUSCA: 산출물을 ingest해 취약점 우선순위·필터에 반영하고, VEX(도달 불가 → not_affected)와 연결.

선행 작업: BomLens 방향 문서(`bomlens-internal/identity-and-direction.md`)와 예고 기능을 확인하고, 공용 산출 포맷·역할 경계를 BomLens 팀과 먼저 협의한다. 고비용 전략 과제이므로 스코프 합의 전에는 착수하지 않는다. 이 항목은 [`bomlens-parity-review.md`](./bomlens-parity-review.md)의 "예고 기능 감시 목록"과 연동해 추적한다.
