# v0.11.0 — TRUSCA로 개명 + 읽기 전용 데모

GA 이후 첫 기능 릴리스입니다. 전체 머신 리더블 체인지로그는
[`CHANGELOG.md`](https://github.com/trustedoss/trusca/blob/main/CHANGELOG.md)에
있습니다. 본 페이지는 사람이 읽는 요약입니다.

## 주요 변경 사항

### 개명 — TrustedOSS Portal은 이제 TRUSCA입니다

제품 이름이 **TRUSCA** — TrustedOSS 이니셔티브의 SCA 도구 — 로 바뀌었습니다.
GitHub 저장소, 컨테이너 이미지, 문서 경로가 새 이름으로 이동했습니다.

- 이미지: `ghcr.io/trustedoss/trusca-backend`, `trusca-backend-worker`,
  `trusca-frontend` (`trusca-` 접두사는 이번 릴리스부터 적용).
- 문서: `https://trustedoss.github.io/trusca/`.
- 내부 식별자(DB 역할, Celery 태스크 이름, 서버 경로)는 의도적으로 기존 이름을
  유지합니다 — 업그레이드에 마이그레이션이 필요 없습니다.

### 공개 읽기 전용 데모

자체 완결형 읽기 전용 데모를 Hetzner 단일 서버에 배포할 수 있습니다. 프로젝트·
스캔·finding이 시드되어 있어 평가자가 데이터를 쓰지 않고도 UI를 살펴볼 수
있습니다. 읽기 전용 모드는 UI에 표시되고 서버 측에서 강제됩니다.

### UI 다듬기 + 보안 강화

타이포그래피 / 모션 / 빈 상태 다듬기 작업(W11~W12)과 외부 검증 캠페인에서 나온
보안 강화 스윕이 이번 릴리스에 포함됐습니다.

## 업그레이드 안내

- 컨테이너 이미지는 새 `trusca-*` 이름으로 게시됩니다. 프로덕션
  `docker-compose.yml`과 Helm 차트는 이미 새 이름을 참조합니다. `IMAGE_TAG`를
  실행 중인 릴리스 버전으로 고정하십시오.
- `0.10.0` 이하 릴리스는 기존 이미지 이름을 그대로 사용하며 영향이 없습니다.
